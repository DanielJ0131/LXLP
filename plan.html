Features

  • Interactive Terminal Emulation: In-browser terminal emulator for hands-on practice.
    • Real-time command execution and feedback.
    • Guided exercises with step-by-step instructions.
    • Sandboxed environment to prevent system damage.
  • Personalized Learning Paths: Adaptive learning paths based on user skill level and goals.
    • Skill assessments to determine starting point.
    • Recommended lessons and exercises based on progress.
    • Customizable learning plans for specific Linux topics.
  • Gamified Learning Challenges: Gamification elements to make learning fun and engaging.
    • Challenges for mastering Linux commands and concepts.
    • Badges and points for achievements.
    • Leaderboards and community rankings.
  • Comprehensive Documentation and Resources: Curated library of Linux documentation and tutorials.
    • Searchable database of Linux commands and concepts.
    • Video tutorials and screencasts.
  • Collaborative Learning Community: Forum and chat for connecting with other Linux learners.
    • Forum for asking questions and sharing tips.
    • Project showcases and code reviews.
  • Virtual Labs and Projects: Virtual environments for practicing real-world Linux scenarios.
    • Simulated server environments for practicing system administration tasks.
    • Guided projects for building Linux-based applications.
    • Customizable virtual labs for advanced learning.

Technologies

  • Frontend: React
  • Backend: Node.js (Express)
  • Database: Mariadb or MongoDB
  • Authentication: JWT
  • Terminal Emulation: xterm.js
  • Virtualization: Docker for virtual labs

Database Contents

  • Users
    • Columns: id, name, email, password, image
  • Posts
    • Columns: postId, userId, content, status, likes, dislikes
  • Comments
    • Columns: commentId, postId, userId, content, likes

Project Structure

Inspired by Mikael's Project Structure

  • Model: Defines the data structure and interacts with the database.
  • View: Manages the user interface and presentation logic.
  • Controller: Handles user input and updates models and views accordingly.
  • Service:Encapsulates database connections
  • Middleware: Handles request and response processing. Error handling such as 404, 500
  • Route: Defines the endpoints for handling HTTP requests and mapping them to appropriate controllers.
  • Config: Centralized configuration for environment variables and application settings.
  • Session Options: Manages session settings for user authentication and state management.
